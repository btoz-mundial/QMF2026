# Runtime Card Philosophy V2

## Objetivo

Este documento define la filosofía oficial de runtime cards del sistema analytics.

Las runtime cards existen para:

- generar narrativa competitiva
- destacar momentos relevantes
- enriquecer timeline_race
- facilitar lectura rápida
- incentivar conversación entre usuarios
- transformar analytics en historias deportivas entendibles

Las runtime cards NO existen para:

- alterar scoring oficial
- inventar narrativa falsa
- exagerar resultados
- reemplazar leaderboard oficial
- generar “casino hype”

---

# Filosofía General

Las runtime cards deben sentirse:

- deportivas
- dinámicas
- inteligentes
- sociales
- modernas

NO deben sentirse:

- sportsbook
- tragamonedas
- clickbait
- esports exagerado
- dashboard corporativo

---

# Principio Fundamental

Toda runtime card debe derivarse de:

- analytics oficiales
- snapshots oficiales
- runtime insights válidos
- deltas reales
- cambios verificables

NO se permite:

- narrativa inventada
- exageración artificial
- inferencias mágicas
- storytelling sin soporte real

---

# Runtime Cards como Capa Narrativa

Las runtime cards representan:

- momentos
- tendencias
- momentum
- estabilidad
- presión competitiva
- evolución del torneo

NO representan:

- scoring oficial
- skill absoluto
- predicciones garantizadas
- rankings permanentes

---

# Filosofía de Timeline Race

Timeline Race es el principal motor narrativo de runtime cards.

Permite detectar:

- remontadas
- caídas
- liderazgo
- consistencia
- presión competitiva
- cambios históricos

Las runtime cards deben aprovechar esta evolución histórica para generar narrativa viva del torneo.

---

# Categorías Oficiales

## Momentum

Detecta:

- ascensos
- caídas
- crecimiento reciente
- pérdida de ritmo

Ejemplos:
- Mayor Remontada
- En Ascenso
- Caída Fuerte

---

## Consistencia

Detecta:

- estabilidad
- permanencia competitiva
- resistencia en rankings

Ejemplos:
- Muro de Acero
- Siempre en la Pelea

---

## Dominio

Detecta:

- liderazgo sostenido
- control del leaderboard
- dominio competitivo

Ejemplos:
- Líder Sólido
- Dominio Total

---

## Supervivencia

Detecta:

- picks vivos
- supervivencia de campeón
- brackets activos

Ejemplos:
- Sigue Vivo
- Última Esperanza

---

## Precisión

Detecta:

- especialización competitiva
- precisión destacada
- dominio de categorías específicas

Ejemplos:
- Killer Eliminations
- Ice Cold
- Table Master

---

## Eficiencia

Detecta:

- aprovechamiento de scoring
- maximización de puntos

Ejemplos:
- Point Hunter
- Eficiencia Elite

---

## Comunidad

Detecta:

- picks colectivos
- tendencias sociales
- comportamiento anti-consenso

Ejemplos:
- Mente Colectiva
- Contra la Corriente

---

# Filosofía de Duración

Las runtime cards pueden ser:

| Tipo | Duración |
|---|---|
| temporary | corto plazo |
| snapshot_based | entre snapshots |
| tournament_wide | gran parte del torneo |

NO todas las cards deben durar igual.

---

# Filosofía de Rareza

Las runtime cards más fuertes deben mantenerse relativamente escasas.

Si demasiados usuarios reciben:

- Dominio Total
- Supremo
- Mayor Remontada

la narrativa pierde impacto.

---

# Runtime Cards NO son Badges Permanentes

Las runtime cards representan:

- estado actual
- narrativa reciente
- momento competitivo

Pueden desaparecer o cambiar durante el torneo.

---

# Relación con Archetypes

Las runtime cards:

- destacan eventos y tendencias

Los archetypes:

- resumen identidad competitiva general

Ambos sistemas son complementarios.

---

# Relación con Tiers

Las runtime cards compatibles con tiers pueden utilizar:

| Tier | Formato |
|---|---|
| ⭐ | Gaming Name |
| ⭐⭐ | Gaming Name Elite |
| ⭐⭐⭐ | Gaming Name Supremo |

Pero NO todas las runtime cards requieren tiers.

---

# Filosofía de Copywriting

El texto de runtime cards debe:

- ser corto
- claro
- competitivo
- entendible rápidamente
- fácil de compartir

---

# Ejemplos Correctos

- Subió 8 posiciones en dos snapshots.
- Mantiene el liderato desde semifinales.
- Sigue vivo con un campeón poco común.
- Entró al Top 5 por primera vez.

---

# Evitar

## Exageración falsa

- El genio definitivo del fútbol.
- Predijo el torneo perfecto.
- Imparable matemáticamente.

---

## Casino/Clickbait

- JACKPOT TOTAL
- PICK MILLONARIO
- INSANE WINNER

---

## Corporativo frío

- Variación estadística positiva detectada.
- Optimización predictiva histórica.

---

# Compatibilidad con Activation Rules

Una runtime card NO debe activarse si:

- el analytic base aún no es stable
- el sample size es insuficiente
- existe alta volatilidad
- el insight aún no es interpretable

---

# Filosofía Visual Implícita

Aunque este documento NO controla UI:

Las runtime cards deben sentirse:

- deportivas
- limpias
- modernas
- dinámicas
- sociales

NO:
- casino
- neon overload
- sportsbook clutter
- fantasy exagerado

---

# Objetivo Final

Las runtime cards deben ayudar a que la quiniela se sienta como:

“una competencia futbolera inteligente entre amigos donde los analytics generan historias, rivalidades y momentos memorables sin perder transparencia ni credibilidad.”
