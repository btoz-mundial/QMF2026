# Naming Conventions — v2

## JSON Naming

All runtime payloads MUST use:

snake_case

## Approved Examples

- match_id
- home_team
- away_team
- home_score
- away_score
- advance
- total_points
- display_name
- exact_goals

## Forbidden

- camelCase
- PascalCase
- semantic aliases
- duplicated meanings

## File Naming

Contracts:
*.contract.v2.md

Registries:
*.registry.v2.json

Schemas:
*.schema.v2.json

Rules:
*.rules.v2.json
