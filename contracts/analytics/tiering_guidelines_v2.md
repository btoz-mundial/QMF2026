# Tiering Guidelines V2

## Objetivo

Este documento define la filosofía oficial de tiers para analytics, archetypes y runtime insights.

El sistema de tiers existe para:

- destacar desempeño competitivo
- crear identidad narrativa
- reforzar engagement
- facilitar lectura rápida
- incentivar rivalidad sana entre usuarios

Los tiers NO existen para:

- reemplazar scoring oficial
- exagerar desempeño
- inflar métricas débiles
- convertir la quiniela en casino
- crear jerarquías artificiales

---

# Filosofía General

Los tiers deben sentirse:

- competitivos
- deportivos
- modernos
- claros
- memorables

NO deben sentirse:

- casino-style
- MMORPG extremo
- esports exagerado
- infantiles
- corporativos

---

# Estructura Oficial de Tiers

Todos los analytics compatibles con tiers utilizan la siguiente estructura:

| Nivel | Formato |
|---|---|
| ⭐ | Gaming Name |
| ⭐⭐ | Gaming Name Elite |
| ⭐⭐⭐ | Gaming Name Supremo |

---

# Filosofía de Cada Tier

## ⭐ Tier Base

Representa:

- desempeño competitivo válido
- resultados positivos consistentes
- analytics destacados

Debe sentirse:

- alcanzable
- satisfactorio
- competitivo

NO debe sentirse:
- automático
- común
- irrelevante

---

## ⭐⭐ Elite

Representa:

- desempeño claramente superior
- consistencia fuerte
- diferenciación competitiva real

Debe sentirse:

- difícil de conseguir
- respetado
- importante

Elite NO debe inflarse excesivamente.

---

## ⭐⭐⭐ Supremo

Representa:

- desempeño excepcional
- resultados poco comunes
- dominio claro dentro del analytic

Debe sentirse:

- raro
- memorable
- prestigioso

Supremo debe mantenerse relativamente escaso.

---

# Flexibilidad del Prototipo 2026

Durante la edición prototipo 2026:

- los thresholds pueden ajustarse dinámicamente
- los percentiles pueden evolucionar
- el frontend puede adaptar distribución de tiers
- Claude puede ayudar a interpretar desempeño relativo

Esto permite:

- observar comportamiento real
- calibrar dificultad
- detectar inflación
- validar engagement
- refinar narrativa

---

# Restricciones Importantes

Aunque exista flexibilidad:

## NO se permite:

- asignar tiers aleatoriamente
- regalar Supremo masivamente
- ignorar analytics oficiales
- ignorar activation rules
- exagerar métricas pequeñas
- inflar tiers por engagement artificial

---

# Filosofía de Asignación

La asignación debe considerar:

- percentiles relativos
- distribución real de usuarios
- sample size válido
- estabilidad estadística
- contexto competitivo del torneo

---

# Compatibilidad con Analytics

Los tiers deben aplicarse principalmente a analytics:

| Analytic | Compatibilidad |
|---|---|
| precision_general | alta |
| precision_avance | alta |
| precision_marcadores_exactos | alta |
| precision_tabla | alta |
| eficiencia_de_puntos | alta |
| consistencia_ranking | media |
| timeline_race | alta |

---

# Analytics con Precaución

Analytics altamente volátiles pueden requerir restricciones temporales.

Ejemplos:

- eliminatorias tempranas
- samples pequeños
- snapshots insuficientes
- rankings inestables

---

# Relación con Activation Rules

Un analytic NO debe asignar tiers si:

- aún no alcanza estado stable
- el sample size es insuficiente
- existe alta volatilidad
- el confidence tier es bajo

---

# Relación con Archetypes

Los tiers NO reemplazan archetypes.

Los archetypes representan:
- identidad competitiva
- estilo observable
- comportamiento general

Los tiers representan:
- nivel relativo de desempeño

---

# Relación con Runtime Insights

Los runtime insights pueden:

- reforzar tiers existentes
- destacar momentos especiales
- explicar cambios competitivos

Pero NO deben:

- alterar scoring
- crear tiers falsos
- ignorar analytics oficiales

---

# Filosofía de Escasez

Los tiers altos deben mantenerse relativamente escasos.

Si demasiados usuarios reciben:
- Elite
- Supremo

el sistema pierde:

- impacto
- identidad
- emoción competitiva
- valor narrativo

---

# Filosofía Visual Implícita

Aunque este documento NO controla UI:

Los tiers deben sentirse:

- deportivos
- elegantes
- modernos
- competitivos

NO:
- tragamonedas
- casino
- “hyper gaming”
- fantasy exagerado

---

# Filosofía de Timeline Race

Timeline Race es uno de los principales motores de tiers dinámicos.

Puede generar:

- ascensos
- caídas
- remontadas
- liderazgo
- estabilidad competitiva

Esto permite crear narrativa viva sin alterar scoring oficial.

---

# Objetivo Final

El sistema de tiers debe ayudar a que la quiniela se sienta como:

“una competencia futbolera inteligente entre amigos donde el desempeño destacado se reconoce de forma clara, narrativa y transparente sin perder credibilidad matemática.”
